# AI Coding Practices:

* **DRY**: Don't repeat yourself. Extract repeated logic into consolidated code.  If viable. use subclassing
* **Magic Numbers**: Avoid. Use named constants.  
* **Comment**: Use comments when logic is complex.  Use comments at the top of files for the whole module.  Use comments for non-trivial function and class headers
* **Surgical Updates**: Perform minimal, precise changes. Do not refactor unrelated code.  Keep focused. Don't do stuff that wasn't asked for.
* **Don't Spin** If you can't fix something after a few attempts, stop and ask for advice.  
* **No HACKS** Don't use hacky solutions just to get desired results.
* **Answer Questions, don't just modify code** If the user asks a question or requests an opinion, it's for a reason. Give an answer, don't change things.
